In this chapter you add two time-aware capabilities. With `iii-cron` you run functions on a schedule:
a nightly sweep of expired links and a daily top-links digest. With `iii-stream` you push every click
to subscribers in real time, so the browser you build next can show a live counter.

## Add the workers

This chapter uses two new workers. Add both now:

```bash
iii worker add iii-cron
iii worker add iii-stream
```

## Give links an expiry

A sweep needs something to sweep, so let links carry an optional expiry. Add an `expires_at` column to
the `links` table in `ensureSchema` — only the SQL string changes:

```typescript src/index.ts {5}
await worker.trigger({
  function_id: 'database::execute',
  payload: {
    db: DB,
    sql: 'CREATE TABLE IF NOT EXISTS links (code TEXT PRIMARY KEY, url TEXT NOT NULL, created_at TEXT NOT NULL, expires_at TEXT)',
  },
})
```

Accept `expires_at` in `link::create` and store it (it stays `null` for links that never expire). The
existing body stays put; the signature, SQL, and params each pick up the new column:

```typescript src/index.ts {3,9,10}
worker.registerFunction(
  'link::create',
  async (payload: { url: string; code?: string; expires_at?: string }) => {
    const code = payload.code ?? makeCode()
    await worker.trigger({
      function_id: 'database::execute',
      payload: {
        db: DB,
        sql: 'INSERT INTO links (code, url, created_at, expires_at) VALUES (?, ?, ?, ?)',
        params: [code, payload.url, new Date().toISOString(), payload.expires_at ?? null],
      },
    })
    await worker.trigger({
      function_id: 'state::set',
      payload: { scope: 'links', key: code, value: { url: payload.url } },
    })
    await publish('link.created', { code, url: payload.url })
    logger.info('link created', { code, url: payload.url })
    return { code, url: payload.url }
  },
)
```

## Sweep expired links nightly

Write a function that deletes expired links from the database and evicts them from the cache, then
bind it to a `cron` trigger. The expression has **seven** fields starting with **seconds** and ending
with **year** (`0 0 3 * * * *` is 03:00 every day, any year):

```typescript src/index.ts
worker.registerFunction('link::sweep_expired', async () => {
  const now = new Date().toISOString()
  const { rows } = await worker.trigger<
    { db: string; sql: string; params: string[] },
    { rows: Array<{ code: string }> }
  >({
    function_id: 'database::query',
    payload: { db: DB, sql: 'SELECT code FROM links WHERE expires_at IS NOT NULL AND expires_at < ?', params: [now] },
  })
  for (const { code } of rows) {
    await worker.trigger({ function_id: 'state::delete', payload: { scope: 'links', key: code } })
  }
  await worker.trigger({
    function_id: 'database::execute',
    payload: { db: DB, sql: 'DELETE FROM links WHERE expires_at IS NOT NULL AND expires_at < ?', params: [now] },
  })
  logger.info('swept expired links', { count: rows.length })
  return { swept: rows.length }
})

worker.registerTrigger({
  type: 'cron',
  function_id: 'link::sweep_expired',
  // Production: 03:00 every day, any year.
  // For testing, swap in the commented every-3-minutes expression below.
  config: { expression: '0 0 3 * * * *' },
  // config: { expression: '0 */3 * * * * *' }, // every 3 minutes — for testing
})
```

The cron schedule runs it nightly, but it is an ordinary function, so you can also run it on demand
to test. Swap the commented expression in if you'd rather watch it fire on a few-minute cadence.

## Build a daily digest

A second scheduled job ranks the most-clicked links and logs them:

```typescript src/index.ts
worker.registerFunction('link::daily_digest', async () => {
  const { rows } = await worker.trigger<
    { db: string; sql: string },
    { rows: Array<{ code: string; clicks: number }> }
  >({
    function_id: 'database::query',
    payload: {
      db: DB,
      sql: 'SELECT code, COUNT(*) AS clicks FROM clicks GROUP BY code ORDER BY clicks DESC LIMIT 5',
    },
  })
  logger.info('daily digest: top links', { top: rows })
  return { top: rows }
})

worker.registerTrigger({
  type: 'cron',
  function_id: 'link::daily_digest',
  // Production: 09:00 every day, any year.
  config: { expression: '0 0 9 * * * *' },
  // config: { expression: '0 */3 * * * * *' }, // every 3 minutes — for testing
})
```

## Stream live clicks

`iii-stream` carries data that subscribers receive over a WebSocket the moment it changes. Publish
each click to a `clicks` stream from `link::record_click`, alongside the durable write — only the new
`stream::set` call is added; the surrounding function is unchanged from Chapter 4:

```typescript src/index.ts {10-19}
worker.registerFunction('link::record_click', async (payload: { code: string; clicked_at: string }) => {
  await worker.trigger({
    function_id: 'database::execute',
    payload: {
      db: DB,
      sql: 'INSERT INTO clicks (code, clicked_at) VALUES (?, ?)',
      params: [payload.code, payload.clicked_at],
    },
  })
  const clickId = `${Date.now()}-${Math.random().toString(36).slice(2, 8)}`
  await worker.trigger({
    function_id: 'stream::set',
    payload: {
      stream_name: 'clicks',
      group_id: 'all',
      item_id: clickId,
      data: { code: payload.code, clicked_at: payload.clicked_at },
    },
  })
  return { recorded: true }
})
```

A `stream::set` does two things: it stores the item, and it broadcasts the change to every WebSocket
subscribed to that stream and group. The browser you build in the next chapter subscribes to
`clicks`/`all` and counts those broadcasts live.

## See it work

Create one link that already expired and one that does not, then run the sweep:

```bash
iii trigger link::create url=https://old.example code=old expires_at=2000-01-01T00:00:00Z
iii trigger link::create url=https://keep.example code=keep
iii trigger link::sweep_expired
```

```json
{ "swept": 1 }
```

The expired link is gone from both the database and the cache, while the other still resolves:

```bash
iii trigger link::resolve code=old
iii trigger link::resolve code=keep
```

```json
{ "url": null }
{ "url": "https://keep.example" }
```

Follow the surviving link a few times, then read the digest and the click stream:

```bash
for n in 1 2 3; do curl -s -o /dev/null http://127.0.0.1:3111/s/keep; done
iii trigger link::daily_digest
iii trigger stream::list stream_name=clicks group_id=all
```

Clicks are queued (Chapter 4), so the count catches up after the queue drains. If the digest
shows fewer than 3 right after the redirects, wait a second and re-run it:

```json
{ "top": [{ "code": "keep", "clicks": 3 }] }
```

The clicks are in the `clicks` stream too, ready for a subscriber.

## Conclusion

Linkly now maintains itself on a schedule (expired links swept nightly, a daily digest of the busiest
links) and emits every click to a live stream. Next, in

and subscribes to that click stream for a live counter.
